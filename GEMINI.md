# プロジェクト進捗状況 (2026-05-06)

## 概要
夫婦間で生活費を共有・精算するスマホ向けWebアプリの開発。

## 実装済み事項

### インフラ・ベース構成
- **モノレポ構成**: `client/` (Next.js), `server/` (Go) のディレクトリ構成を構築。
- **Docker**: MySQL 8.0 と Go サーバーのコンテナ環境を整備。
- **CORS**: フロントエンド (localhost:3000) とバックエンド (localhost:8080) の通信許可設定。

### バックエンド (Go / Gin / GORM)
- **DBモデル**: `users`, `groups`, `receipts`, `settlements` のテーブル定義。
- **認証 & セキュリティ**: 
  - ユーザー登録・ログイン・プロフィール管理。
  - **認証エラーの統一**: ユーザー未存在時の 404 を 401 (Unauthorized) に統一し、フロントエンドでの自動ログアウトを確実化。
- **レシート管理**: 
  - CRUD操作、Gemini AI 解析 (`POST /api/receipts/analyze`)。
  - **権限管理**: 登録者本人のみ更新・削除が可能となるようガードを実装。
  - **精算対象月**: 購入日とは独立した精算月 (`settlement_year`, `settlement_month`) を保持。
- **グループ管理**: 作成、所属一覧取得、メンバー招待、メンバー削除。
- **サマリー・精算**: 
  - **高度な精算ロジック**: 押し切り型の精算を廃止し、任意金額の記録と履歴管理 (`Amount` フィールド追加) に対応。
  - **履歴取得**: 誰がいついくら精算したかの履歴を返却。

### フロントエンド (Next.js / Tailwind CSS / shadcn/ui)
- **認証・ナビゲーション**:
  - **BottomNav のコンポーネント化**: 認証状態に応じた表示制御を実装。未ログイン時は非表示。
  - **ログインUX改善**: 401エラー時の自動遷移を見直し、ログイン失敗メッセージが消えないよう修正。
- **メイン機能**: 
  - **レシート一覧/詳細**: 支払者のニックネームを表示。
  - **編集制限**: 本人以外のレシートを読み取り専用（編集・削除不可）に。
  - **精算対象月の入力**: レシート登録・編集時に精算月を指定可能に。
- **サマリー・精算**: 
  - **精算バランス計算**: `レシート合計 - 精算済み履歴` による正確な残高表示。
  - **部分精算対応**: 金額を入力して精算を記録する UI を構築。残高以上の入力は不可。
  - **操作権限**: お金を払う側（バランスがプラス）の人だけが精算ボタンを押せるよう制限。
  - **精算履歴表示**: 月ごとの精算アクションを一覧表示。
- **グループ管理**: 設定画面から新規グループの作成が可能に。
