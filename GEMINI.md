# プロジェクト進捗状況 (2026-05-17)

## 概要
夫婦間で生活費を共有・精算するスマホ向け Web アプリの開発。

## 実装済み事項

### インフラ・ベース構成
- **モノレポ構成**: `client/` (Next.js), `server/` (Go) のディレクトリ構成を構築。
- **Docker**: MySQL 8.0 と Go サーバーのコンテナ環境を整備。
- **クライアントの Docker 化**: Next.js (Standalone モード) を Docker 化し、`compose.yml` に統合。
- **外部アクセス・HMR設定**: 
  - スマホ実機テスト用に `next.config.ts` でホストおよび `allowedDevOrigins` を許可。
  - バックエンド側で CORS 設定を拡張。
- **環境変数管理**: `.env` ファイルによる API キー管理を導入し、`compose.yml` からの参照設定を完了。
- **Dockerfile 最適化**: 
  - Go サーバーにマルチステージビルドを導入し、イメージサイズを大幅に削減（約320MB → 85MB）。
  - `.dockerignore` の整備によりビルド効率を向上。

### バックエンド (Go / Gin / GORM)
- **DBモデル**: `users`, `groups`, `receipts`, `settlements` のテーブル定義。
- **認証 & セキュリティ**: 
  - ユーザー登録・ログイン・プロフィール管理。
  - **認証エラーの統一**: ユーザー未存在時の 404 を 401 に統一。
- **レシート管理**: 
  - CRUD 操作、Gemini AI 解析（API キー設定済み）。
  - **年月フィルタ**: `GET /api/receipts` で `year`, `month` による絞り込みに対応。
- **サマリー・精算**: 
  - 高度な精算ロジック、部分精算、履歴管理。
  - **バリデーション**: 精算額が 1 円以上であることを必須化。

### フロントエンド (Next.js / Tailwind CSS / sonner)
- **UI/UX 改善**:
  - **月切り替えボタン**: 矢印アイコンを濃く・大きくし、視認性と操作性を向上。
  - **状態保持**: レシート一覧の年月選択を URL パラメータ化。詳細画面から戻った際にも選択していた月を維持。
  - **通知機能**: `sonner` によるトースト通知。
- **認証・ナビゲーション**:
  - **BottomNav**: 認証状態に応じた表示制御。
- **レシート管理**: 
  - **一覧画面**: 支払者ごとの色分け、精算方法の日本語ラベル表示。
  - **詳細画面**: AI 解析、本人限定の編集・削除、精算済みロック。
