# プロジェクト進捗状況 (2026-05-04)

## 概要
夫婦間で生活費を共有・精算するスマホ向けWebアプリの開発。

## 実装済み事項

### インフラ・ベース構成
- **モノレポ構成**: `client/` (Next.js), `server/` (Go) のディレクトリ構成を構築。
- **Docker**: MySQL 8.0 と Go サーバーのコンテナ環境を整備。
- **CORS**: フロントエンド (localhost:3000) とバックエンド (localhost:8080) の通信許可設定。

### バックエンド (Go / Gin / GORM)
- **DBモデル**: `users`, `groups`, `receipts`, `settlements` のテーブル定義。`groups` に `owner_id` (NOT NULL) を追加。
- **認証 & セキュリティ**: 
  - ユーザー登録・ログイン・プロフィール管理。
  - **厳格なトークン検証**: `AuthMiddleware` にてリクエストごとにユーザーの実在性をDB確認するよう強化。
- **レシート管理**: CRUD操作、Gemini AI 解析 (`POST /api/receipts/analyze`) の実装。
- **グループ管理**: 作成、所属一覧取得、メンバー招待、メンバー削除。
- **サマリー・精算**: 月次集計 API（精算日時表示対応）、精算確定機能。

### フロントエンド (Next.js / Tailwind CSS / shadcn/ui)
- **基本レイアウト**: モバイル最適化、ボトムナビゲーション。
- **動的なグループハンドリング**:
  - 固定の `group_id=1` を廃止し、所属グループを動的に取得して利用するよう全画面（一覧・登録・サマリー）を刷新。
  - グループ未所属時の案内画面（Empty State）を追加。
- **メイン機能画面**: レシート一覧、レシート登録（AI解析連携）、詳細（編集・削除）。
- **サマリー画面**: 月次集計・支払いバランス表示・精算確定機能（精算日時の表示対応）。
- **設定画面 (`/profile`)**: アカウント情報の変更、グループ管理（招待・削除）、ログアウト。
- **エラーハンドリング & UX**:
  - サーバー切断時の専用エラー画面 (`ServerError`) と Next.js オーバーレイの抑制。
  - トークン無効/DBリセット時の自動ログアウト・ログイン画面遷移。
  - ネットワークエラー判定の強化 (Chrome/Firefox/Safari対応)。

## 次にやるべきこと

1. **UX/UIのブラッシュアップ**
   - 保存・削除・精算成功時のトースト通知（Sonner 等の導入）。

2. **テストと検証**
   - 複数ユーザー（夫婦間）での招待〜登録〜精算のフルサイクル確認。
   - 実際のスマホブラウザからの実機操作確認。
3. **デプロイ準備**
   - 環境変数の整理と本番環境へのデプロイ手順の整備。
