# プロジェクト進捗状況 (2026-05-09)

## 概要
夫婦間で生活費を共有・精算するスマホ向け Web アプリの開発。

## 実装済み事項

### インフラ・ベース構成
- **モノレポ構成**: `client/` (Next.js), `server/` (Go) のディレクトリ構成を構築。
- **Docker**: MySQL 8.0 と Go サーバーのコンテナ環境を整備。
- **クライアントの Docker 化**: Next.js (Standalone モード) を Docker 化し、`compose.yml` に統合。ビルド引数で API URL を切り替え可能。
- **外部アクセス設定**: スマホ実機テスト用に `next.config.ts` でホストを許可し、サーバー側で CORS 設定を拡張。

### バックエンド (Go / Gin / GORM)
- **DBモデル**: `users`, `groups`, `receipts`, `settlements` のテーブル定義。
- **認証 & セキュリティ**: 
  - ユーザー登録・ログイン・プロフィール管理。
  - **認証エラーの統一**: ユーザー未存在時の 404 を 401 に統一。
- **レシート管理**: 
  - CRUD 操作、Gemini AI 解析。
  - **年月フィルタ**: `GET /api/receipts` で `year`, `month` による絞り込みに対応。
  - **権限管理**: 登録者本人のみ更新・削除が可能。
  - **データロック**: 精算済み（`settled_at` あり）のレシートは編集・削除を禁止。
  - **バリデーション**: 金額が 1 円以上であることを必須化。
- **グループ管理**: 
  - 作成、所属一覧、メンバー招待・削除。
  - **管理機能**: グループ名の変更、およびグループの削除（関連データの一括削除を含む）を実装。
- **サマリー・精算**: 
  - **高度な精算ロジック**: 任意金額の記録と履歴管理。
  - **ロジック改善**: 「自分が10割負担」のバグ修正および精算方法のコード値化（half, self, other）。
  - **バリデーション**: 精算額が 1 円以上であることを必須化。

### フロントエンド (Next.js / Tailwind CSS / sonner)
- **通知機能 (sonner)**:
  - `alert()` を廃止し、洗練されたトースト通知を導入。通知位置を画面下部（80vh）に調整。
- **認証・ナビゲーション**:
  - **BottomNav**: 認証状態に応じた表示制御。
  - **ログイン/新規登録**: エラー表示をインラインからトースト通知に改善。
- **レシート管理**: 
  - **一覧画面**: 
    - 年月を選択して表示するレシートを切り替え可能に。
    - **ユーザー別の色分け**: 支払者ごとにバッジの色を分けることで視認性を向上。
    - **表示ロジック修正**: 精算方法を日本語ラベルで表示。
  - **詳細画面**: 
    - AI 解析、本人限定の編集・削除。
    - **精算済みロック**: 精算済みの場合はフォームを読み取り専用にし、ステータスを表示。
  - **入力バリデーション**: 金額の負数/ゼロ入力をフロントエンドでもガード。
- **サマリー・精算**: 
  - 精算バランス計算、部分精算、精算履歴表示。
  - **バリデーション**: 残高を超える精算額の入力を制限。
- **グループ管理**: 
  - 新規作成に加え、管理者による名前変更・削除機能を追加。
