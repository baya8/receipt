# 概要

夫婦間で生活費などを共有して精算できるWebアプリを作る。
アプリはスマホで主に使うことを想定している。（Android / IPhone）


# 技術仕様

## 技術選定

- フロント
  - Next.js (TypeScript)
  - スタイリング: Tailwind CSS
  - UIコンポーネント: shadcn/ui
- バックエンド
  - Go (Gin)
  - ORM: GORM
- データベース
  - MySQL
- AI
  - Gemini API (レシート画像識別)

## フォルダ構成

```text
receipt/
├── client/          # フロントエンド (Next.js)
├── server/          # バックエンド (Go)
├── docker/          # MySQL等のインフラ設定
├── docker-compose.yml
└── README.md
```


# アプリ仕様

PoCとして、まずは以下の機能を実装したい。

## アカウント登録

- メールアドレス、パスワード、ニックネームを登録
- メールアドレスはワンタイムコードで認証
- パスワードはDBに不可逆暗号化して保存

## レシート一覧

- 参加しているグループの入力されたレシート一覧が表示される
- 一覧上は日付・品名・金額・精算方法が簡略的に見れるものとする
- 一覧上の明細をタップ／クリックすると明細の詳細画面へ遷移

## レシート登録

以下の内容を手入力するか、撮影したレシート画像をgemini AIで画像識別し、必要な項目を自動設定する

- 購入日
  - レシートの日付
- お店
  - 購入したお店
- 品名
  - レシートの大まかな内容
- 金額
  - レシート記載の合計金額
- 支払者
  - だれが支払ったか
  - ここは自分だったり、同じグループ内のだれか、でもよい
- 精算方法
  - 以下の内容から選択
    - 自分が10割負担
    - 折半（グループ内で割り勘）
    - 全額相手負担（特定のＮさんが10割負担）
    - 負担割合手入力
      - グループ内の人間で誰がいくら負担するかを手で入力していく
      - この機能はPoCの段階では未実装でよい


## レシート詳細画面

- 入力されたレシートを参照する画面。
- 詳細画面からは、自分の登録した明細であれば編集が可能
- グループ内で管理者権限を持つ人であれば、ほかの人が登録した明細も編集可能

## サマリー画面

月ごとに、誰が誰にいくら支払うか、を確認できる画面。
また精算済みの金額は、精算済みとわかるようなUIにする。

- 月の切り替え
- 月ごとの自分の精算額の表示
- 月ごとのほかのメンバの精算額の表示
- 自分の精算について、未精算か精算済みかを記録できる仕組み
- 精算ボタンをタップして精算済みにしたとき、いつ精算したか？が分かるように日付を表示する

## 設定画面

アカウントの情報やグループ管理を行うための画面。
以下のメニューが存在。
- アカウント情報
  - アカウントの情報や変更が行える
- グループ管理
  - 今属しているグループのメンバ一覧が表示される
  - グループ管理者は、グループメンバの招待／削除が可能
- ログアウト
  - ログアウトする
